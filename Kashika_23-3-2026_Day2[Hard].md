This problem ask us to return the smallest positive integer (>=1) that is NOT present in the given array. 
I have first sorted the array so as to arrange them in ascending order. This will ease the process considerately. Then we start a loop with i="0" that adds 1 to itself after every iteration. If the element at i'th element is less than 0 we skip that iteration as we need to find the "positive" missing number. We have initialized another variable 'j' as 1 (which is the smallest positive number). Now we check if the first element of the array is equal to j (which is 1) cuz that is supposed to be the starting point. if the condition is true, then we add 1 to j and move on with the next iteration. Or if the element at i is greater than j, then that deduces that element present does not have j in the array whihc will then become the smallest positive missing element. Hence we return j. This also helps ignore the duplicates (if any) present in the array.
