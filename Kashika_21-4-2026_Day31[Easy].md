This code solves the classic “Climbing Stairs” problem using a sleek, space-efficient dynamic programming approach. Instead of storing all intermediate results, it keeps track of only the last two values, like a traveler remembering just the previous two steps taken. The key insight is that the number of ways to reach step *n* equals the sum of ways to reach steps *n-1* and *n-2*, mirroring the Fibonacci sequence. For small inputs where *n ≤ 2*, the answer is simply *n*. The variables `prev` and `curr` represent the number of ways to reach the two preceding steps, and they are updated iteratively in a loop from 3 to *n*. At each step, a temporary variable stores their sum, then the values shift forward. By the end, `curr` holds the final answer. This approach runs in linear time and constant space, making it both fast and memory-friendly.
