This problem is a level up from the "Easy" reverse Linked List problem. Here we are asked to reverse a LL but in groups of 'k'. Means that we have to pick up 'k' simultaneous elements from a LL and then reverse them and put them back in there place.
The code here is simple and to the point. First we assign an iterator 'temp'. Then we check whether there are enough elements to reverse in group of 'k'. If not, then we simply return the LL. If yes, then we reassign temp to the head of LL and assign prev to NULL. Then we start a loop that reverses links between nodes one at a time. This point is crucial. First we record the next element of temp in a node. Then we assign the prev as the next node to temp. We then move prev to temp's position and temp to next's position.
After this loop ends, we call the function again so that we may check if another part of linked list maybe reversed. After reversion is completed, we return prev as it is the current element.
