The problem of day 11 is apparently a simpler version of day 10's merging sorted array (which i didn't attempt cuz it was complex even though i did have a little idea how it would be done). So the idea was with me how to solve it. And by limiting the number of lists, it became easier. So first we make a new dummy node that will be the start of our merged list. Them we will assign new node to the head of the LLs so as to not change their heads. We will also take a iterator for the dummy node/list. We then initiate a loof that runs till either of the list do not end. If the loop we check whether the value at the given iterator is greater than or less than the otehr list's. According to it, we add the lesser value to the dummy/tail, then move the iterator and the tail so as to point it towards the end of the merged list. Once either of the lists ends, we check whether the other list has any elements and add them to the merged list as it is (cuz the lists are also sorted). Then we return the next node to dummy which is the head of the merged sorted list.
